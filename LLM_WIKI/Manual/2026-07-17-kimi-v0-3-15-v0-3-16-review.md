Kimi raised one important potential issue: the strict boolean selector for
`local_summary_enabled` might serialize under an unknown frontend selector
name instead of Home Assistant's native `boolean` type.

## Resolution

Direct inspection of the installed Home Assistant Core 2026.6.4 and 2026.7.0
implementations showed that `BooleanSelector` defines
`selector_type = "boolean"`, subclasses inherit that attribute, and
`Selector.serialize()` uses it. Both versions already serialized the HausmanHub
subclass as `{"selector": {"boolean": {}}}`.

The contract was still made explicit in `StrictBooleanSelector`. The local
adapter test now requires the same selector type, and the disposable Core
harness converts the real options `data_schema` through Home Assistant's
frontend serializer and requires exactly one `local_summary_enabled` field
with the native boolean selector.
