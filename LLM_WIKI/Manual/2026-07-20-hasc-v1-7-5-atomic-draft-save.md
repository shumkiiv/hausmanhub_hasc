# HASC 1.7.5: атомарное сохранение климатического контура

Дата: 2026-07-20. Пункт плана: 16 из 50.

## Что готово

`POST /api/hausman_hub/v1/admin/climate-drafts/save` принимает точный черновик
HASC. Внутри одной блокировки HASC перечитывает актуальное состояние, повторяет
полную проверку и только после этого разрешает приватные привязки устройств.

Сохраняются вместе:

- реестр комнат и выбранных климатических устройств;
- их типы и проверенные возможности;
- название и режим климатического контура;
- температура, влажность и характер работы каждой комнаты;
- новое служебное состояние готовности для сохранённого реестра.

Сборка использует существующую модель HASC и двигатель
`existing_climate_core`. Это продолжение уже реализованного климатического
контура, а не отдельная параллельная настройка.

## Защиты

- Маршрут доступен только локальному администратору Home Assistant.
- Старый номер снимка даёт HTTP 409 до любой записи.
- Изменённый или заблокированный черновик даёт HTTP 400 до любой записи.
- Запись реестра, контура и служебного состояния защищена общим откатом.
- При ошибке восстанавливается предыдущая рабочая конфигурация.
- Ответ не содержит приватных ID и прямо сообщает `commands_sent: false`.
- Сохранение не отправляет команды устройствам и не требует перезапуска.
