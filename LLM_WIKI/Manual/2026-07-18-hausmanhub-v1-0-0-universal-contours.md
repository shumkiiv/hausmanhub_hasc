# HausmanHub 1.0.0: универсальные контуры и существующий климатический алгоритм

Дата: 2026-07-18.

## Решение

HausmanHub 1.x является фасадом и платформой автоматических контуров. Климат —
первый тип. Обычный пользователь выбирает контур, комнаты, устройства и
параметры. Технические реестр, bridge, shadow, canary и старый нативный preview
не являются продуктовым интерфейсом и находятся в дополнительных настройках.

Климатический алгоритм не переписывается. Контур `climate` использует движок
`existing_climate_core`, публично названный `hausman-climate` v1. Он сохраняет
ответственность за профили, авто/ручной режим, cooldown, safety, authority,
физическое выполнение и обратную связь. HausmanHub отвечает за выбор и стабильное
описание контура, понятные параметры, статус и Android boundary.

## Реализация 1.0.0

- Общий домен содержит версионный registry контуров, тип, lifecycle и engine.
- Климатический контур назначает несколько комнат и несколько устройств,
  хранит температуру, влажность и стратегию.
- Обычный мастер читает свежий Climate API state, показывает только названия
  комнат/устройств и использует непрозрачные временные selector tokens.
- Выбранные source-managed устройства получают стабильные публичные HausmanHub ID;
  повторный HA control endpoint не требуется, потому что приватной связью уже
  владеет climate-core.
- Приватный climate registry и contour registry сохраняются атомарно с
  откатом первой записи при ошибке второй.
- Публичный `GET /api/hausman_hub/v1/contours` использует контракт
  `hausman-hub-contours` v1 и не выдаёт source ID, entity ID или адрес.
- `automatic_active` истинно только при свежем state, авто-режиме engine,
  готовой authority, доступных устройствах и совпадающих целях.
- Disabled bridge не делает сетевого запроса. Создание контура сохраняет
  внутреннее подключение в shadow/read-only. Climate POST отсутствует.

## Граница и следующий шаг

Параметры версии 1.0.0 являются желаемой декларацией и проверяются против
фактических engine targets. Они ещё не синхронизируются в engine. Несовпадение
даёт `attention` и не подтверждает автоматическую работу. Отключение контура в
HausmanHub не отправляет внешнему двигателю команду off.

Следующий шаг 1.x — явно подтверждаемая синхронизация room target, strategy и
auto mode через закрытые типизированные команды, идемпотентную квитанцию,
повторное чтение и честное состояние частичного результата. Затем нужны разные
параметры комнат, расписания/профили и Android UI.

## Проверка

До публикации должны пройти полный локальный набор, release/file-safety,
disposable Core 2026.6.4 и 2026.7.0 и независимый staged review. Живой deploy
ограничивается установкой точной версии через HACS без перезапуска, настройки
контура и физические команды автоматически не выполняются.
