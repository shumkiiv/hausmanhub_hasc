# HausmanHub 1.7.3: административный API черновика климата

Дата: 2026-07-20. Пункт плана: 14 из 50.

## Что готово

- `GET /api/hausman_hub/v1/admin/climate-drafts` отдаёт готовые варианты формы:
  комнаты, устройства, предложенные комнаты, поддерживаемые типы и русские
  названия.
- `POST` на тот же адрес принимает номер снимка, название, режим, комнаты,
  комфортные цели и короткие ссылки устройств.
- Ответ `hausman-hub-climate-draft` v1 нормализован и имеет собственный
  `draft_revision`, но содержит `save_allowed: false` и
  `validation_required: true`.
- Внешние ID источника, объекты Home Assistant и команды не входят ни в один
  публичный ответ.

## Безопасность

Маршрут принимает только обычного локального администратора Home Assistant.
Пользователь планшета и внешний адрес получают отказ. Старый номер состава
устройств даёт HTTP 409. Лишние поля, повтор устройства, перенос в другую
комнату, неподходящий тип и неверные комфортные цели дают HTTP 400.

Оба метода только читают свежий снимок Climate API. Они не сохраняют реестр и
контур, не отправляют команд и не меняют даже оперативные доказательства
готовности теневого режима.
