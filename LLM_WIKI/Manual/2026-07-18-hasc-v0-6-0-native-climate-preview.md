# HASC 0.6.0: первый встроенный климатический расчёт

Дата: 2026-07-18.

## Решение

Климатический контур можно и нужно постепенно строить внутри HASC. Android
остаётся клиентом только HASC. Во время миграции существующий climate-core даёт
текущие показания и остаётся физическим исполнителем, поэтому перенос правил не
требует резкого переключения живого дома.

## Реализованный этап

HASC хранит политику одной комнаты: стабильный HASC ID, температуру от 18 до
28 °C с шагом 0,5 °C и влажность от 30 до 70 % с шагом 5 %. Чистый доменный
расчёт использует свежий снимок и нейтральные зоны ±0,5 °C и ±5 %, после чего
возвращает нагрев, охлаждение, увлажнение, удержание или отсутствие данных.

Расчёт проверяет, зарегистрировано ли доступное оборудование нужного типа.
Устаревшее состояние не используется как основание для действия. Высокая
влажность показывается отдельно, потому что устройство осушения пока не
моделируется.

В настройках Home Assistant появился пятый понятный раздел «Встроенный
климатический контур HASC». Владелец выбирает режим предпросмотра, сохранённую
комнату и две цели. Перед сохранением HASC показывает текущие значения,
решение, доступность оборудования и явную строку «Отправка команд: нет».

## Граница безопасности

Предпросмотр всегда возвращает `execution.mode=preview_only` и
`commands_enabled=false`. Доменный расчёт не содержит транспорта. Runtime
может только прочитать состояние в уже разрешённом режиме проверки; при
выключенной связи он не выполняет I/O. Настройка не меняет реестр, не создаёт
Home Assistant service call и не выполняет Climate API POST.

Старые config entries не содержат новых полей и интерпретируются как полностью
выключенный встроенный контур. Отключение удаляет сохранённую комнату и цели.

## Публикация

Код опубликован коммитом `a765cc7` и релизом `v0.6.0`. Все 244 локальных теста,
проверки релизного пакета и проверки на Home Assistant Core 2026.6.4/2026.7.0
прошли. Финальное read-only ревью не нашло существенных замечаний. HACS
показывает установленную и последнюю версию `v0.6.0`; для загрузки нового кода
в Home Assistant владельцу остаётся выполнить обычный перезапуск.

## Следующие этапы

1. Добавить нативные Home Assistant bindings для датчиков температуры и
   влажности и сравнить их с переходным Climate API.
2. Сформировать HASC-owned typed desired-action plan без исполнения.
3. Добавить cooldown, ручной режим, конфликтующие потребности и безопасный
   rollback.
4. Опубликовать решение и цели в следующей версии Android-контракта.
5. Только по отдельному разрешению провести нативный canary одной комнаты с
   физическим подтверждением и сохранённым старым исполнителем для отката.
