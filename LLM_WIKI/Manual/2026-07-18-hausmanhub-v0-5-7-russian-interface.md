# HausmanHub 0.5.7: понятный русский интерфейс

Дата: 2026-07-18.

## Решение

Пользовательский интерфейс Home Assistant и главная страница репозитория HausmanHub
ведутся на понятном русском языке. Названия режимов должны объяснять действие,
а не показывать внутренний английский код.

## Причина смешанного языка

Русский файл перевода содержал технические слова `canary`, `shadow`,
`rollout`, `input_boolean`, `source ID`, `control entity`, `scope` и `owner`.
Кроме того, постоянные списки передавали Home Assistant явную английскую
подпись, равную внутреннему значению. Такая подпись могла перекрывать русский
перевод пункта. Динамические результаты также подставляли в русские предложения
коды `ready`, `blocked`, `true`, `set_room_target` и коды причин.

## Изменения

- Русские названия, описания и ошибки мастера полностью переписаны простым
  языком.
- Постоянные списки передают только внутренние значения и ключ перевода. Явная
  английская подпись для них запрещена тестом.
- Динамические статусы, причины, действия и логические значения превращаются в
  русские фразы до показа. Название комнаты берётся из сохранённого списка.
- Неизвестный внутренний код не показывается пользователю и заменяется общей
  безопасной фразой.
- README и названия шагов GitHub Actions переведены на русский. Краткое описание
  репозитория GitHub должно быть обновлено при публикации 0.5.7.

## Граница безопасности

Русификация не меняет климатический контракт, права, режим моста, реестр или
команды. Живое развёртывание обязано оставить климатический контур выключенным.

## Проверка

Перед независимым ревью прошёл полный набор из 231 теста. Отдельные временные
Home Assistant Core 2026.6.4 и 2026.7.0 подтвердили новую форму постоянных
списков и русские результаты мастера. Публикация, установка через HACS и
перезапуск Home Assistant пока не выполнены.
