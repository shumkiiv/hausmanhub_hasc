# HausmanHub: название продукта и установка через HACS

Дата: 2026-07-20.

## Решение

Название продукта и интеграции Home Assistant — **HausmanHub**.

**HACS** — отдельный каталог Home Assistant, через который пользователь
устанавливает и обновляет HausmanHub. Эти названия нельзя использовать как
синонимы.

Внутренний домен Home Assistant остаётся `hausman_hub`. Его переименование
сломало бы связь с уже сохранённой записью интеграции, поэтому исправляется
только ошибочное публичное название, а совместимый внутренний ключ сохраняется.

## Что исправлено

- имя интеграции, интерфейс, README и русские инструкции;
- адрес GitHub на `shumkiiv/hausmanhub_hacs`;
- публичные имена контрактов на `hausman-hub-*`;
- схемы, синтетические примеры, тесты и служебная документация;
- названия внутренних каталогов примеров и проверки контрактов;
- отдельная пошаговая установка через пользовательский репозиторий HACS.

Проверка выпуска ищет возврат старого временного имени, старого префикса
контрактов или прежнего адреса репозитория и останавливает сборку.

## Проверка

- 355 локальных тестов и полная проверка выпуска прошли;
- временные Home Assistant Core 2026.6.4 и 2026.7.0 загрузили интеграцию и
  завершили полный жизненный цикл без ошибки;
- Kimi `k2p7` не начал проверку из-за ошибки сервера
  `err_7c5c5f07` (сессия `ses_08191e6cdffeZFO3U7kFPu1jNi`), а Kimi `k3` не
  успел завершить большой просмотр в ограниченное время;
- независимая резервная проверка OpenCode на модели
  `opencode/deepseek-v4-flash-free` завершилась статусом PASS в сессии
  `ses_0818e4910ffeNoKwAj0QEiB030`. Она отдельно подтвердила сохранение домена
  `hausman_hub`, постоянных `unique_id`, согласованность схем и примеров и
  правильное разделение HausmanHub/HACS.
