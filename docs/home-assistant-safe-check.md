# Безопасная проверка в Home Assistant

Эта проверка показывает, что HASC установился и остаётся безопасной
интеграцией без управления домом. В версии 0.3.13 она умеет составлять только
общую сводку из чисел: без названий, точных идентификаторов и показаний.

## Перед началом

Установлена Home Assistant Core 2026.6.4 или новее. HASC должен быть добавлен
в HACS как пользовательский репозиторий `shumkiiv/hausmanhub_hasc` с типом
«Integration».

Если HACS показывает старую версию или иконка не появилась:

1. В HACS выберите «Update information».
2. Откройте HausMan Hub HASC и выберите «Redownload».
3. Перезапустите Home Assistant.

## Проверка

1. Откройте «Настройки» → «Устройства и службы» → «Интеграции».
2. Убедитесь, что у HausMan Hub HASC есть иконка дома с маленьким узлом в
   центре.
3. Нажмите «Добавить интеграцию», найдите **HausMan Hub HASC** и выберите
   режим **Только чтение**.
4. Убедитесь, что интеграция добавилась без ошибки.
5. Откройте HASC и убедитесь, что появились ровно девять строк с общими
   числами. Не должно быть устройства HASC, кнопок управления или служб.
6. Для дополнительной проверки сводки откройте меню с тремя точками на
   карточке HASC и выберите «Скачать диагностические данные». В разделе
   `home_summary` должны быть только те же общие числа. Подробнее — в документе
   [«Безопасная сводка о доме»](read-only-home-summary.md).

## Необязательный пользователь для просмотра

Обычные девять строк HASC и проверка через диагностический файл работают без
отдельного пользователя. Специальный пользователь «только просмотр» нужен
только человеку или программе, которым по желанию нужна дополнительная
локальная страница с теми же девятью числами.

Этот пользователь не даёт доступ HASC: интеграция не получает и не хранит его
пароль, ключ или адрес для подключения к Home Assistant. Она только разово
проверяет, что запрос пришёл из домашней сети, и ничего об этом не сохраняет.
Не создавайте для пользователя права администратора и не передавайте пароль
или ключ в чат. Условия описаны в [руководстве по локальному
доступу](read-only-local-access.md).

Можно открыть параметры интеграции и переключить только между «Только
чтение» и «Проверка без изменений». В начале рекомендуется оставлять «Только
чтение». После сохранения такого выбора HASC применит его сама: перезапустится
только HASC, а не Home Assistant и не дом.

## Что прислать для проверки

Достаточно снимка страницы интеграций после добавления HASC и, если иконка
ещё не появилась, снимка карточки HACS после «Redownload». Если нужна помощь
со сводкой, можно вручную написать только числа из `home_summary`.

Не присылайте архив диагностики, файлы конфигурации, адрес Home Assistant,
пароли, ключи или список устройств.

## Что означает ошибка

- «Requires Home Assistant 2026.6.4» означает, что HACS скачал старую версию.
  Выполните три шага из раздела «Перед началом».
- Ошибка после выбора `proxy` или другого режима, кроме двух показанных,
  означает, что защита сработала правильно.
- Если HASC выключена, временно остановлена, удалена или у неё две настройки,
  диагностический файл и ранее открытая страница сводки сообщают только, что
  сводка недоступна. Включите одну правильную настройку обычными средствами
  Home Assistant; HASC сама ничего не исправляет.
- Если при повреждённой настройке экран выбора режима показывает «Только
  чтение», это защита. Простое открытие экрана ничего не исправляет и не
  сохраняет: сначала вручную проверьте настройку обычными средствами Home
  Assistant.
- Если сохранённая настройка будет повреждена, пока HASC уже работает, HASC
  сразу закроет девять чисел, диагностический файл и дополнительную локальную
  страницу. Она не будет читать данные дома, пока настройка не будет исправлена
  вручную и HASC не будет перезагружена обычной кнопкой Home Assistant.
- Если HASC предлагает управление устройством, остановите проверку и
  сообщите об этом: текущая версия этого делать не должна.
