# Как выпустить обновление HASC

Эта короткая памятка нужна только тогда, когда меняется сама интеграция HASC
или её файл установки `hacs.json`. Обычное изменение инструкции, тестов или
проверок не требует нового обновления в HACS.

## Перед выпуском

1. Поднимите номер версии в `custom_components/hausman_hub/manifest.json`.
   Например, после `0.2.0` используйте `0.2.1`.
2. Добавьте сверху в `CHANGELOG.md` простое описание изменений этой версии.
3. Запустите `python3 tools/check_local_release.py`. Он остановится, если
   интеграция изменилась, а номер версии остался прежним. Он также проверит
   подготовленный набор файлов для HACS: саму интеграцию, две локализации,
   иконку, короткие сведения HACS, лицензию и понятное описание версии.
   Проверка читает только подготовленные Git-файлы и не подключается к дому.
4. Для изменения кода получите независимый просмотр Kimi и исправьте все
   замечания. Kimi должен проверить окончательный текущий набор изменений до
   того, как изменение будет считаться завершённым или будут выполнены коммит,
   отправка, выпуск, развёртывание или публикация. Если Kimi временно
   недоступен, запишите другой независимый просмотр. Он позволяет продолжать
   любые изменения внутри границ HASC: код, тесты, документацию, местные
   проверки и исправления. Он не разрешает коммит, отправку, выпуск, развёртывание,
   публикацию или новые права. Исключение для изменения только документации
   действует лишь когда в наборе нет кода; в смешанном наборе действует
   финальная проверка Kimi.
5. Сохраните и отправьте изменения в GitHub. Дождитесь успешной проверки
   GitHub.

## Публикация и установка

1. На GitHub создайте опубликованный выпуск с меткой `v` и тем же номером,
   например `v0.2.1`. Выпуск должен указывать ровно на проверенное изменение.
2. В HACS откройте страницу **HASC — управление домом**, откройте меню с тремя точками
   и выберите **Обновить сведения**.
3. Если HACS показывает новую версию, установите её и перезапустите Home
   Assistant.

HACS показывает удобный выбор версий, когда в репозитории есть опубликованные
выпуски. Для добавленного вручную репозитория команда **Обновить сведения**
запрашивает свежие сведения у GitHub. См. официальные пояснения HACS:
[публикация интеграций](https://www.hacs.xyz/docs/publish/integration/) и
[обновление сведений](https://www.hacs.xyz/docs/faq/data_sources/).

## Что остаётся неизменным

Выпуск сам не включает управление: он может содержать только заранее явно
одобренную область. В версии 0.5.0 прежний `input_boolean` canary и новый
климатический мост выключены по умолчанию. Климат запускается отдельно:
приватный реестр, shadow, одна authority-ready canary-комната и откат в
`disabled`. Generic proxy и прямые caller-selected services остаются
запрещены. Не прикладывайте к выпуску ключи, адреса, реальные source/entity ID,
названия устройств, диагностические архивы или данные дома.
