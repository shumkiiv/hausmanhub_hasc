# Совместимость HausmanHub с моделями Android

Статус для HausmanHub 1.6.9: автоматическая проверка данных добавлена. Android-проект
не изменялся и остаётся только источником для чтения.

## Что проверяется

Скрипт `tools/check_android_compatibility.py` берёт синтетический ответ
`GET /api/hausman_hub/v1/home` из репозитория HausmanHub и проверяет, что:

- идентификаторы и названия остаются обычными строками;
- время и номер состояния помещаются в тип Android `Long`, а номер состояния
  также точно представим обычным числом JSON;
- температура и влажность представимы конечными числами `Double` или `null`;
- каждая комната ссылается на понятные климатические устройства;
- тип каждого устройства однозначно переводится в домен Android;
- у каждого показанного действия есть название, описание, разрешение и
  понятное русское объяснение блокировки;
- для установки температуры и выключения комнаты приложение может собрать
  запрос, проходящий строгую схему действий HausmanHub.

Эта проверка входит в общий локальный выпускной прогон и не читает Android-
репозиторий. Поэтому она одинаково работает локально и в GitHub Actions.

## Чего эта проверка пока не означает

В существующем Android-приложении ещё нет законченного сетевого декодера
контракта дома HausmanHub v12. Проверка подтверждает, что публичные значения HausmanHub
укладываются в уже используемые приложением типы, но не утверждает, что экран
планшета уже подключён к этим маршрутам.

Живое подключение приложения выполняется в отдельной ветке разработки Android.
Со стороны HausmanHub дальнейшие изменения API должны сохранять эту проверку зелёной.
