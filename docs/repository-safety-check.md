# Проверка безопасности файлов перед публикацией

Эта проверка помогает не отправить в GitHub личные данные дома по ошибке.
Она читает только файлы этого репозитория и ничего не подключает к Home
Assistant, Node-RED или интернету.

## Что она ищет

Проверка останавливается при обнаружении типичных опасных файлов: ключей,
паролей, настроек Home Assistant, резервных копий и файлов Node-RED. Также она
распознаёт несколько распространённых видов ключей прямо внутри текстовых
файлов.

Это дополнительная защита, а не замена внимательной проверки человеком.
Названия устройств, адрес дома, диагностический файл и ключ доступа всё равно
нельзя добавлять в репозиторий.

## Как запускать

Проверить все уже опубликованные файлы:

```bash
python3 tools/check_repository_boundary.py
```

Проверить только файлы, подготовленные для ближайшей отправки:

```bash
python3 tools/check_repository_boundary.py --staged
```

Если проверка не прошла, не отправляйте изменения. Уберите личный файл из
подготовленных изменений, а ключ при необходимости сразу замените.
