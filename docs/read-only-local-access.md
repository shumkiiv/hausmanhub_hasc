# Локальный доступ только к сводке HASC

Дата решения: 2026-07-14.

Владелец явно разрешил HASC локальный доступ только к девяти общим числам
сводки дома. Это не разрешение на управление домом, доступ к списку устройств
или доступ из интернета.

Обычные девять строк HASC в интерфейсе Home Assistant работают без отдельного
пользователя. Отдельный пользователь нужен только человеку или программе,
которым по желанию нужна дополнительная локальная страница с теми же девятью
числами.

## Что появилось

В версии `0.2.0` HASC добавляет одну необязательную защищённую страницу внутри
Home Assistant для такого просмотра. Она умеет только отвечать на запрос
чтения и возвращает девять чисел из [безопасной
сводки](read-only-home-summary.md).

Перед ответом HASC проверяет все условия:

1. Запрос пришёл из домашней сети или с самого Home Assistant.
2. Пользователь Home Assistant вошёл с отдельной ролью «только просмотр».
3. Этот пользователь не администратор, не системный пользователь и не состоит
   одновременно в какой-либо другой группе прав.
4. Конфигурация HASC по-прежнему находится в одном из двух безопасных
   режимов: «Только чтение» или «Проверка без изменений».

Если не выполнено хотя бы одно условие, HASC не выдаёт даже сводку.

## Чего этот доступ не даёт

- Он не показывает названия, идентификаторы, показания, историю или события.
- Он не принимает команды и не содержит кнопок управления.
- Эта отдельная страница не создаёт дополнительных устройств, сущностей, служб
  или автоматизаций. Девять строк HASC с общими числами описаны в
  [безопасной сводке](read-only-home-summary.md).
- Он не обращается к Node-RED, Climate, Automation или Smart Home Center.
- Он не отправляет данные наружу. HASC не получает и не хранит пароль, ключ
  или адрес для подключения к Home Assistant. Он только разово проверяет, что
  запрос пришёл из домашней сети, и ничего об этом не сохраняет.
- Proxy и прямое выполнение остаются запрещены.

## Как включать безопасно

Создавайте отдельного пользователя для просмотра только если действительно
нужна дополнительная локальная страница. Для обычных девяти строк HASC и
проверки через диагностику он не нужен.

Этот пользователь нужен тому, кто открывает страницу, а не HASC. HASC не
входит под этой учётной записью и не получает её пароль, ключ или адрес для
подключения к Home Assistant. Если страница нужна, пользователь не должен быть
администратором, не должен быть привязан к человеку или присутствию и должен
иметь только встроенную роль «только просмотр».

Не передавайте пароль, адрес Home Assistant или ключ доступа в чат и не
кладите их в этот репозиторий. Если дополнительная страница не нужна, ничего
больше делать не надо: обычные девять строк HASC и диагностика уже работают.
Если интерфейс Home Assistant не предлагает роль «только просмотр», не
настраивайте эту дополнительную страницу и не редактируйте внутренние файлы
Home Assistant вручную. Обычные девять строк и диагностика всё равно останутся
доступны.
