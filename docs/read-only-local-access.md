# Локальный доступ только к сводке HausmanHub

Дата решения: 2026-07-14.

Владелец явно разрешил HausmanHub локальный доступ только к девяти общим числам
сводки дома. Это не разрешение на управление домом, доступ к списку устройств
или доступ из интернета.

Эта граница относится только к GET-странице сводки. Отдельный opt-in canary
версии 0.4.0 не использует эту страницу и описан в [своём
контракте](canary-input-boolean-control.md).

Обычные девять строк HausmanHub в интерфейсе Home Assistant работают без отдельного
пользователя. Отдельный пользователь нужен только человеку или программе,
которым по желанию нужна дополнительная локальная страница с теми же девятью
числами.

## Что появилось

В версии `0.2.0` HausmanHub добавляет одну необязательную защищённую страницу внутри
Home Assistant для такого просмотра. Она умеет только отвечать на запрос
чтения и возвращает девять чисел из [безопасной
сводки](read-only-home-summary.md).

Перед ответом HausmanHub проверяет все условия:

1. Запрос пришёл с самого Home Assistant — с адреса `127.x.x.x` или `::1` —
   либо с обычного адреса домашней сети: `10.x.x.x`, `172.16.x.x`–`172.31.x.x`,
   `192.168.x.x` или с домашнего IPv6-адреса, начинающегося с `fc` или `fd`.
   Пустые, служебные, тестовые, временные, внешние и другие специальные адреса
   HausmanHub не принимает. Иногда IPv4-адрес записывается внутри IPv6 как
   `::ffff:…`; HausmanHub примет такую запись только если адрес внутри неё сам
   относится к одному из перечисленных IPv4-адресов, включая `127.x.x.x`.
2. Пользователь Home Assistant вошёл с отдельной ролью «только просмотр».
3. Этот пользователь не администратор, не системный пользователь и не состоит
   одновременно в какой-либо другой группе прав.
4. Конфигурация HausmanHub по-прежнему находится в одном из двух безопасных
   режимов: «Только чтение» или «Проверка без изменений».

Если не выполнено хотя бы одно условие, HausmanHub не выдаёт даже сводку.
У страницы есть только один точный адрес: вариант с лишней косой чертой в
конце или с дописанными данными после вопросительного знака не становится
вторым способом получить сводку.

Если сам подсчёт девяти чисел неожиданно не получается, страница также отвечает
только «сводка недоступна». Она не показывает техническую причину и не выдаёт
часть чисел.

## Чего этот доступ не даёт

- Он не показывает названия, идентификаторы, показания, историю или события.
- Он не принимает команды и не содержит кнопок управления.
- Эта отдельная страница не создаёт дополнительных устройств, сущностей, служб
  или автоматизаций. Девять строк HausmanHub с общими числами описаны в
  [безопасной сводке](read-only-home-summary.md).
- Он не обращается к Node-RED, Climate, Automation или Smart Home Center.
- Он не отправляет данные наружу. HausmanHub не получает и не хранит пароль, ключ
  или адрес для подключения к Home Assistant. Он только разово проверяет, что
  запрос пришёл из домашней сети, и ничего об этом не сохраняет.
- Каждый ответ, который формирует эта дополнительная страница, содержит
  правило «не сохранять». Поэтому браузер не должен показывать старую копию
  девяти чисел из своей памяти после отключения HausmanHub или потери прав.
- Proxy и общее выполнение над физическими устройствами остаются запрещены.

## Как включать безопасно

Создавайте отдельного пользователя для просмотра только если действительно
нужна дополнительная локальная страница. Для обычных девяти строк HausmanHub и
проверки через диагностику он не нужен.

Этот пользователь нужен тому, кто открывает страницу, а не HausmanHub. HausmanHub не
входит под этой учётной записью и не получает её пароль, ключ или адрес для
подключения к Home Assistant. Если страница нужна, пользователь не должен быть
администратором, не должен быть привязан к человеку или присутствию и должен
иметь только встроенную роль «только просмотр».

Не передавайте пароль, адрес Home Assistant или ключ доступа в чат и не
кладите их в этот репозиторий. Если дополнительная страница не нужна, ничего
больше делать не надо: обычные девять строк HausmanHub и диагностика уже работают.
Если интерфейс Home Assistant не предлагает роль «только просмотр», не
настраивайте эту дополнительную страницу и не редактируйте внутренние файлы
Home Assistant вручную. Обычные девять строк и диагностика всё равно останутся
доступны.

## Как закрыть дополнительную страницу

Откройте параметры HausmanHub и выключите пункт «Дополнительная локальная страница».
После сохранения HausmanHub перезапустится сама. Девять обычных строк HausmanHub и файл
диагностики останутся доступны. Если старый адрес страницы был открыт раньше,
он ответит только «сводка недоступна» и не запустит подсчёт именно для этого
запроса. Девять обычных строк продолжают обновляться из тех же уже разрешённых
общих чисел: иначе они перестали бы быть актуальными.
После следующего полного запуска Home Assistant эта закрытая страница вообще
не будет добавлена.

Этот выбор не управляет домом, не создаёт нового адреса и не меняет данные,
которые HausmanHub может читать. Повторное включение возвращает только ту же
необязательную страницу с теми же девятью общими числами.
